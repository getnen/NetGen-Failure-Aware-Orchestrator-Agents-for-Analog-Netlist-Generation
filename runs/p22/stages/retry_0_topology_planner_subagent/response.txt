**Topology Summary**

Use a 5 V single-supply transistor-level RC phase-shift oscillator with output node exactly `Vout`. The oscillator core should be an inverting, high-gain MOS analog amplifier biased around mid-supply, followed by a three-section RC phase-shift ladder that feeds back from `Vout` to the amplifier input. The RC ladder supplies approximately 180 degrees of phase shift at the oscillation frequency; the inverting gain stage supplies the remaining 180 degrees.

Target oscillation frequency should be in the approximate 5 kHz to 20 kHz range so `.tran 1u 5m` shows many visible cycles.

**Device/Function List**

- `VDD` supply source from `VDD` to ground, value 5 V.
- Mid-supply reference generator producing `VREF` near 2.5 V, preferably a resistor divider from `VDD` to ground with optional bypass capacitor.
- MOS transistor-level inverting gain stage:
  - Output node must be `Vout`.
  - Use real `NMOS`/`PMOS` devices only.
  - Bulk terminals tied to each device source.
  - Gain must comfortably exceed the passive RC ladder attenuation, typically requiring voltage gain greater than about 29 at startup.
- Three RC ladder sections from `Vout` back to the amplifier input.
- Bias resistors from RC/amplifier input nodes to `VREF` so no node floats in DC.
- Startup assistance using `.ic` on one or more internal RC ladder/input nodes.

**Bias/Reference Strategy**

Bias the amplifier input and output operating point around `VREF = 2.5 V`. The gain stage should settle near mid-rail in `.op`, with enough headroom for `Vout` to swing above and below 2.5 V without immediately clipping.

If using a single-ended common-source style gain block, bias the gate/input node near a valid operating point around `VREF`, and use active/current-source or resistive loading to obtain high gain.

If using a transistor-level differential gain block, keep explicit input nodes such as `Vinp` and `Vinn`. Bias one input at `VREF`, feed the final RC ladder node into the other input, and ensure polarity gives net loop inversion through the amplifier.

**Signal Or Feedback Path**

Feedback path should be explicit:

`Vout -> RC section 1 -> RC section 2 -> RC section 3 -> amplifier input`

Use three cascaded RC sections referenced to `VREF`, not floating capacitive-only nodes. Example node roles:

- `Vout`: amplifier output and oscillator output.
- `n1`: first RC ladder internal phase node.
- `n2`: second RC ladder internal phase node.
- `n3` or amplifier input node: final phase-shifted feedback node.
- `VREF`: AC reference / virtual ground for ladder shunt elements and amplifier bias.

The final RC node drives the inverting amplifier input. Internal ladder nodes should show progressively shifted, lower-amplitude sine-like waveforms relative to `Vout`.

**SPICE/Netlist Constraints**

- Final filename: `p22.cir`.
- Include node `Vout` exactly.
- Include exactly:
  - `.tran 1u 5m`
- Use primitive SPICE devices: MOSFETs, resistors, capacitors, voltage sources.
- Do not use behavioral sources, controlled sources, ideal op-amp macros, ideal limiters, or inverter-style rail-switching as the oscillator core.
- Every internal RC node must have a DC path to `VREF` or ground.
- Add `.ic` such as a small offset on the final RC node or amplifier input node to break perfect symmetry at startup.
- Keep `Vout` unclamped unless normal transistor saturation naturally limits amplitude.

**Failure Risks To Avoid**

- Insufficient amplifier gain; the three-section RC ladder attenuates strongly, so loop gain must exceed unity at startup.
- Floating RC ladder nodes caused by capacitors without DC bias paths.
- Biasing the amplifier so `Vout` sits near a rail instead of near 2.5 V.
- Accidentally building a hard CMOS inverter oscillator instead of an analog high-gain stage.
- Using clamp diodes or ideal limiters on `Vout`, which can mask bad oscillator design.
- Wrong feedback polarity: the RC ladder plus amplifier must produce total loop phase near 360 degrees.
- R/C values that oscillate too slowly or too fast for `.tran 1u 5m`.
- Perfectly symmetric initial conditions that prevent startup in transient simulation.
