=== TASK ===
Type: PLL
Description: This Phase-Locked Loop (PLL) circuit synchronizes an output oscillator with an external reference signal, achieving phase alignment and frequency control. It consists of three key components: a phase detector, a voltage-controlled oscillator (VCO), and a loop filter, all implemented using MOSFET technology for precise control and stability. The phase detector compares the phase of the input reference signal with the output of the VCO. Any phase difference generates an error signal, which indicates whether the VCO should speed up or slow down. This error signal then passes through a loop filter, which smooths out rapid changes, generating a stable control voltage for the VCO. The voltage-controlled oscillator adjusts its frequency based on this control voltage, gradually aligning its phase with the reference signal. In this MOSFET-based design, each component is optimized for integration and low-power operation. The phase detector uses MOSFETs configured to produce an accurate and fast response to phase differences. The loop filter ensures a stable and low-noise control voltage, while the VCO adjusts the output frequency in response to minor fluctuations in the control voltage, thereby locking onto the reference signal. This PLL circuit provides reliable phase synchronization suitable for applications in communication systems, frequency synthesis, and signal processing.
Inputs: clk_ref
Outputs: clk_p

Objective:
Implement a primitive-MOS PLL that locks the oscillator output clk_p to the external reference input clk_ref by generating a phase/frequency error, filtering it into a stable control voltage, and steering a MOS VCO whose tuning range straddles the reference frequency.

Supply convention and operating assumptions:
Use a single 1.8 V supply with nodes VDD = 1.8 V and VSS = 0 V.
Drive the required input node clk_ref explicitly with an independent pulse source referenced to VSS; a reasonable default is 0 V to 1.8 V, 50% duty cycle, with a nominal 10 MHz reference unless the downstream implementation chooses another clearly documented target.
Use MOSFET bulk tied to source for discrete idealized primitive devices unless a shared-substrate process assumption is explicitly introduced and documented.
Provide a mid-supply bias reference near 0.9 V only where needed for analog bias/current-setting stages; do not leave control or detector nodes floating.

Topology:
Use one contiguous closed-loop path:
1. Phase/frequency detector comparing clk_ref against the fed-back VCO clock.
2. MOS charge-pump or equivalent up/down transconductance stage that converts detector decisions into sourced or sunk current on the loop-filter control node.
3. Loop filter/control-storage network producing VCTRL.
4. Current-starved ring VCO whose oscillation frequency is materially controlled by VCTRL.
5. Output buffer exposing the actual controlled oscillator output as clk_p, and feed that same clock back directly to the detector input.

Preferred implementation:
Use a compact MOS phase-frequency detector rather than an XOR-only detector, because the PLL must capture frequency error as well as phase error. A simple two-latch/two-reset PFD is acceptable if implemented contiguously and minimally, with actively driven UP and DOWN nodes and a real reset path.
Follow the PFD with a charge pump made from matched PMOS/NMOS current-steering devices charging/discharging the loop-filter node. Avoid using large leak resistors as the main state-holding mechanism.
Use a passive loop filter with at least one real storage capacitor on VCTRL and a modest resistor in series or parallel as needed to add damping and reduce ripple. The filter must smooth UP/DOWN pulses into a stable analog control voltage.
Use a 3-stage or 5-stage current-starved ring oscillator. The control node VCTRL must set the current available to the delay cells so the oscillation frequency clearly increases or decreases with VCTRL. Avoid strong always-on inverter branches that let the oscillator free-run almost independently of VCTRL.
Buffer the ring output to produce a rail-to-rail clk_p suitable both for observation and for feedback into the detector.

Required devices:
Primitive NMOS and PMOS transistors for:
phase/frequency detector latches and reset logic,
charge-pump current sources/sinks and switching devices,
current-starved inverter delay cells in the VCO,
output buffer/inverter stages,
bias mirrors or bias-setting transistors if needed.
Passive components:
loop-filter capacitor(s),
modest loop-filter resistor(s),
small startup or damping passives only if clearly justified.
Independent sources:
VDD supply source,

Key interconnectivity:

clk_p must be the buffered VCO output and must also be the feedback clock returned to the PFD comparison input.
The PFD must produce UP and DOWN control actions that directly drive a contiguous charge/discharge path into the loop-filter control node VCTRL.
VCTRL must connect directly to the VCO control devices that starve or boost the ring-delay current.

Biasing and sizing assumptions:
Choose modest initial MOS sizing, favoring symmetric inverters in the ring and matched pump devices to reduce static phase offset.
Bias the charge pump so VCTRL can settle over a meaningful tuning span, for example roughly 0.3 V to 1.5 V on a 1.8 V supply.
Size the VCO so its free-running center frequency at mid-control is near the assumed clk_ref frequency, with the full tuning range spanning below and above that frequency to allow capture.

Expected behavior:

During lock, clk_p should converge to the clk_ref frequency with a bounded phase offset and reduced control-voltage ripple after filtering.
The transient result should show capture behavior first, then near-aligned periodic waveforms for clk_ref and clk_p.
