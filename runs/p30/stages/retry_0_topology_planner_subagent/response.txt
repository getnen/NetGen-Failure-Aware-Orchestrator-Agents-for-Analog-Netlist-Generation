**Topology summary**

Use a compact closed-loop MOS PLL with this path:

`clk_ref` -> PFD -> UP/DN -> charge pump -> loop filter / `VCTRL` -> current-starved ring VCO -> output buffer -> `clk_p` -> direct feedback to PFD.

Recommended implementation:
- Phase/frequency detector: minimal two-state PFD built from MOS latches or edge-triggered set/reset stages, with explicit UP and DN outputs and an active reset path when both edges have occurred.
- Charge pump: matched PMOS source branch and NMOS sink branch that charge/discharge `VCTRL` according to UP/DN.
- Loop filter: one real capacitor on `VCTRL` plus a modest damping resistor/capacitor arrangement if needed to reduce ripple and peaking.
- VCO: 3-stage current-starved ring oscillator preferred for simplicity. `VCTRL` must directly control the starvation devices in each delay cell.
- Output stage: one or two CMOS inverters buffering one ring node to produce rail-to-rail `clk_p`; feed this same buffered clock back to the PFD.

**Device/function list**

- NMOS/PMOS latch or pulse-memory devices for PFD state generation.
- NMOS/PMOS reset devices for the PFD common reset path.
- PMOS current-source / switch devices for UP charging action.
- NMOS current-sink / switch devices for DN discharging action.
- Loop-filter capacitor on `VCTRL`.
- Optional modest loop-filter resistor for damping or ripple shaping.
- Three current-starved inverter delay cells for the ring VCO.
- Bias-setting MOS devices or mirrors only if needed to make pump/VCO currents repeatable.
- CMOS output buffer stages generating `clk_p`.
- Independent sources: `VDD`, ground `VSS`, and pulse source driving `clk_ref`.

**Bias/reference strategy**

- Single supply: `VDD = 1.8 V`, `VSS = 0 V`.
- Drive `clk_ref` explicitly with a pulse source at the chosen reference frequency; nominal 10 MHz is acceptable.
- Keep bulk tied to source unless a different substrate assumption is intentionally documented.
- Hold `VCTRL` with a real storage capacitor; do not rely on leakage for detector or filter state definition.
- If analog bias is needed for current mirrors or pump reference current, derive a simple mid-supply related bias near 0.9 V, but do not leave control nodes floating.
- Size the VCO so mid-range `VCTRL` gives a free-running frequency near the target reference, with tuning span both below and above it.

**Signal or feedback path**

- `clk_ref` enters the PFD reference side directly.
- `clk_p` is the buffered VCO output and must also be the only feedback clock sent back to the PFD.
- PFD UP/DN outputs must actively drive the charge-pump switches.
- Charge pump sources/sinks net current into the single inspectable control node `VCTRL`.
- `VCTRL` directly modulates the current-starving transistors in the ring delay path.
- Expected transient behavior:
  - If `clk_p` is slow, UP dominates, `VCTRL` rises, VCO speeds up.
  - If `clk_p` is fast, DN dominates, `VCTRL` falls, VCO slows down.
  - In lock, `clk_p` frequency approaches `clk_ref`, with bounded phase offset and reduced `VCTRL` ripple.

**SPICE/netlist constraints**

- Preserve node names exactly: `clk_ref`, `clk_p`.
- Keep one explicit control node, preferably `VCTRL`.
- No behavioral PLL abstraction, no VCVS/VCCS shortcuts for the main loop.
- Every helper node used later must be actively generated by a present MOS stage.
- Avoid XOR-only phase detection; use a real PFD so frequency error is captured.
- Transient analysis should be long enough to show acquisition and settling; for 10 MHz reference, start around 50 us to 200 us.
- If startup help is needed for the ring, use a small intentional asymmetry or documented initial condition, not an unrealistic artificial driver.

**Failure risks to avoid**

- VCO tuning range entirely above or below the `clk_ref` target, preventing capture.
- Current-starved ring cells with strong uncontrolled inverter paths that bypass `VCTRL` and weaken tuning sensitivity.
- PFD nodes that float or depend on giant leak resistors instead of active set/reset action.
- Charge pump overlap causing static shoot-through or large reference-spur ripple on `VCTRL`.
- Using an internal unbuffered ring node for feedback while exporting a different node as `clk_p`; feedback should come from the same controlled output path.
- Oversized loop-filter resistor/capacitor causing either excessive ripple, very slow lock, or unstable hunting.
- Output buffer loading the ring too heavily and shifting the VCO center frequency; isolate with a dedicated buffer stage.
