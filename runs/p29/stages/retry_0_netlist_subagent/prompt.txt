=== TOPOLOGY PLAN ===
**Topology Summary**

Use a single-supply, transistor-level two-stage MOS op-amp core with explicit `Vinp`, `Vinn`, and `Vout`. Power it from `VDD = 12 V` and `VSS = 0 V`, and create `VREF = 6 V` as the internal AC reference. Close the positive-frequency-selective loop with a classic Wien bridge from `Vout` to `Vinp`: series branch `C1` then matched MOS-triode variable resistor `R1(Vctrl)`, and shunt branch from `Vinp` to `VREF` as matched MOS-triode variable resistor `R2(Vctrl)` in parallel with `C2`. Set the negative-feedback divider from `Vout` to `Vinn` to `VREF` for non-inverting gain slightly above 3 so startup is reliable without diode limiting.

**Device / Function List**

- Op-amp input stage: NMOS differential pair with drains loaded by PMOS current-mirror active load.
- Biasing for input stage: NMOS tail current source, derived from a simple MOS bias mirror.
- Second gain stage: one common-source MOS stage driven by the first stage output, with modest analog swing capability.
- Compensation: one Miller capacitor between second-stage output node and first-stage high-gain node if needed for stable closed-loop analog behavior.
- Output node: `Vout` is the second-stage output and the directly observed oscillator output.
- Wien bridge series arm: `Vout` -> `C1` -> internal bridge node -> MOS triode resistor `R1(Vctrl)` -> `Vinp`, or equivalent ordering as long as it is a single series RC path.
- Wien bridge shunt arm: `Vinp` -> `R2(Vctrl)` to `VREF` and `Vinp` -> `C2` to `VREF`.
- Frequency control devices: matched MOSFETs used only as triode-region variable resistors for `R1(Vctrl)` and `R2(Vctrl)`, optionally each with a small fixed bounding resistor.
- Gain-setting network: resistor from `Vout` to `Vinn` and resistor from `Vinn` to `VREF`.
- `Vin` injection: explicit independent source on node `Vin`, coupled weakly into `Vinp` through a large resistor or small capacitor.
- `Vctrl`: explicit independent source on node `Vctrl`, feeding the common gate-bias network for both triode resistors.
- `VREF` generator: resistor divider from 12 V to 0 V plus decoupling capacitor; optional light buffering only if done with primitive transistors, but keep it simple.

**Bias / Reference Strategy**

Bias the signal path around `VREF = 6 V`, not ground. `Vinp` and `Vinn` should sit near `VREF` at operating point. The differential pair common-mode range must comfortably include `VREF`, and the second stage must allow centered analog swing around this midpoint. Size the op-amp for loop gain above the Wien loss at the nominal oscillation frequency, but not so aggressively that it heavily loads the bridge. Bias `R1(Vctrl)` and `R2(Vctrl)` from `Vctrl` through one simple translation network if needed so both MOS resistors remain in triode over the small-signal swing around `VREF`.

**Signal / Feedback Path**

`Vout` feeds two feedback paths:

- Positive, frequency-selective path: `Vout` through the Wien bridge back to `Vinp`. At the oscillation frequency, the bridge should present about zero phase shift and about one-third attenuation.
- Negative, broadband path: resistive divider from `Vout` to `Vinn` to `VREF`, setting closed-loop non-inverting gain around 3.1 to 3.3.

`Vin` is not part of the main loop; it is only a weak startup injection so the required node exists and is not floating. `Vctrl` modulates only the matched bridge resistors, so frequency moves monotonically mainly through `f ≈ 1 / (2*pi*R(Vctrl)*C)`.

**SPICE / Netlist Constraints**

- Preserve node names exactly: `Vin`, `Vctrl`, `Vout`, `Vinp`, `Vinn`.
- Use primitive MOSFETs, resistors, capacitors, and independent sources only.
- No ideal op-amp macro, no `E/G/F/H/B` controlled-source core, no behavioral resistor, no switches, no diode limiter.
- Keep the op-amp input nodes explicit and electrically meaningful.
- Use one nominal `Vctrl` source value in the deck; frequency variation is for separate external runs, not `.step`.
- Include `.op` and one transient run long enough for startup and settled oscillation.
- Bulk ties should be explicit and consistent with the primitive-device assumption.

**Failure Risks To Avoid**

- Setting closed-loop gain at or below 3 nominally; it may never start.
- Letting gain be far above 3 and relying on rail clipping; that will distort `Vout` and can destabilize the bridge.
- Driving the MOS variable resistors out of triode over the bridge swing; then frequency control becomes nonlinear or broken.
- Mismatching `R1(Vctrl)` and `R2(Vctrl)` or `C1` and `C2`; the Wien condition shifts and startup becomes unreliable.
- Loading `Vinp` with oversized devices or extra branches; this weakens the bridge selectivity.
- Making `VREF` too soft or noisy; bridge centering and amplifier bias will wander.
- Leaving `Vin` floating or coupling it too strongly; either hurts convergence or corrupts steady-state oscillation.
- Hiding multiple feedback ladders or extra buffers around `Vout`; that adds poles and makes transient behavior hard to predict.
